# M3-01 Test Script — Private Trip Details

## Build and family view

- [ ] Build badge reads **M3-01**.
- [ ] The normal family itinerary contains no real confirmation numbers.
- [ ] The day dashboard no longer shows a budget tile.
- [ ] The replacement Weather tile says live forecast is planned for M3-03.

## First-time setup

- [ ] Open **Private Details**.
- [ ] Enter a 4–8 digit numeric PIN.
- [ ] Confirm the PIN.
- [ ] Mismatched PINs show an understandable message.
- [ ] Successful setup opens Organizer Reservation Vault.

## Saving details

- [ ] Open a reservation.
- [ ] Add a confirmation number.
- [ ] Add a ticket URL.
- [ ] Add a booking-document URL.
- [ ] Add a phone number.
- [ ] Add parking/check-in notes.
- [ ] Add a private organizer note.
- [ ] Select Save details.
- [ ] A saved confirmation appears.
- [ ] Reload the app and verify the details remain on that device.

## Links

- [ ] Ticket link opens in a new tab.
- [ ] Booking-document link opens in a new tab.
- [ ] Phone link opens the phone dialer on mobile.

## Locking

- [ ] Select Lock.
- [ ] Private information disappears.
- [ ] Incorrect PIN does not unlock the vault.
- [ ] Correct PIN unlocks it.
- [ ] Private details appear on the applicable daily screen only while unlocked.

## Data controls

- [ ] Remove details for one reservation.
- [ ] Confirm other reservations remain unchanged.
- [ ] Clear all private data.
- [ ] Confirm the vault is empty afterward.

## Device behavior

- [ ] Open the shared family link in another browser/device.
- [ ] Confirm private information is not present there.
- [ ] Confirm public itinerary and reservation statuses still work normally.
